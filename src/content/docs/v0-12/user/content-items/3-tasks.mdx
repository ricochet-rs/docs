---
title: Tasks
---

Tasks are content items that run to completion.
Unlike [apps](/v0-12/user/content-items/2-dynamic-content), which stay running to serve requests, a task executes once and then stops — for example an R script that generates a report, a Python ETL pipeline, or a Quarto document that renders to HTML.

See the [content items overview](/v0-12/user/content-items/1-content-items-overview) for the full list of supported task types.

## Task types

### R

| Content Type | Description |
| ------------ | ----------- |
| `r`          | An R script with the extension `.R` or `.r`. |
| `rmd`        | An [R Markdown](https://rmarkdown.rstudio.com/) document. Can be served as a [static site](/v0-12/user/tasks/3-static-sites). |
| `quarto-r`   | A [Quarto](https://quarto.org/) document using an R runtime. Can be served as a [static site](/v0-12/user/tasks/3-static-sites). |

### Python

| Content Type | Description |
| ------------ | ----------- |
| `python`     | A Python script with the extension `.py`. |
| `quarto-py`  | A Quarto document using a Python runtime. Can be served as a [static site](/v0-12/user/tasks/3-static-sites). |

### Julia

| Content Type | Description |
| ------------ | ----------- |
| `julia`      | A Julia script. |
| `quarto-jl`  | A Quarto document using a Julia runtime. Can be served as a [static site](/v0-12/user/tasks/3-static-sites). |

## Working with tasks

- **[Invoking tasks](/v0-12/user/tasks/1-invoke)** — run tasks on-demand via the UI, CLI, or REST API
- **[Scheduling tasks](/v0-12/user/tasks/2-scheduling)** — run tasks on a recurring schedule
- **[Static sites](/v0-12/user/tasks/3-static-sites)** — serve static HTML generated by tasks like R Markdown or Quarto
