Deploy your first app in minutes using our public trial instance at [try.ricochet.rs](https://try.ricochet.rs).
Note that the trial instance is wiped weekly.
Do not use it for production purposes.

<Steps>

1. Create an account at [try.ricochet.rs](https://try.ricochet.rs)

    Or, you can login to your private instance.

2. Install the ricochet CLI

    The [ricochet CLI](https://github.com/ricochet-rs/cli) is used to deploy items to your ricochet server.

    {/* prettier-ignore */}
    <Tabs>
    <TabItem label="Mac / Linux">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/ricochet-rs/cli/main/install.sh | sh
    ```
    </TabItem>
    <TabItem label="Windows">
    ```bash
    curl.exe -fsSL https://raw.githubusercontent.com/ricochet-rs/cli/main/install.ps1 -o install.ps1; .\install.ps1; Remove-Item install.ps1
    ```
    </TabItem>
    </Tabs>

3. Login to the ricochet instance

    The `-S` flag is used to specify the server url you are logging into.

    ```bash
    ricochet login -S https://try.ricochet.rs
    ```

4. Deploy a Shiny app

    First, clone an example Shiny app and navigate to it:

    {/* prettier-ignore */}
    <Tabs>
    <TabItem label="Mac / Linux">
    ```bash
    git clone https://github.com/rstudio/shiny-examples.git /tmp/shiny-examples
    cd /tmp/shiny-examples/050-kmeans-example/
    ```
    </TabItem>
    <TabItem label="Windows">
    ```bash
    git clone https://github.com/rstudio/shiny-examples.git
    cd shiny-examples/050-kmeans-example/
    ```
    </TabItem>
    </Tabs>

    Now deploy the app:

    ```bash
    ricochet deploy
    ```

    You'll be prompted to create a `_ricochet.toml` file.
    This file is required for deploying to ricochet and tracks your item's configuration.

    Enter `yes`.

5. Select the language

    Choose **R** from the language options (R, Julia, or Python).

    ```
    ? Choose a language â€º R
    ```

6. Select the content type

    Choose **Shiny** from the content type options.
    You can type "sh" to filter the list.

    ```
    ? Choose content type â€º Shiny
    ```

7. Specify the entrypoint

    The entrypoint is the file or folder that starts your app.
    For this example, select **app.R**.

    ```
    ? Select Shiny app entrypoint â€º app.R
    ```

8. Give the item a name

    Enter a display name for your app.
    This name appears in the ricochet UI.
    ```
    ? Content item name â€º My First Shiny App
    ```

9. Set the item's visibility

    Choose who can access your app:
    - **Private**: Only you and specified users
    - **Internal**: Only logged-in users
    - **External**: Public-facing, no authentication

    For this quickstart, select **External**.

    ```
    ? Item visibility â€º External
    ```

    Once the deployment is complete, the app can be viewed.

</Steps>
